# ☁️ Cloud Service Models (IaaS, PaaS & SaaS)

Cloud Service Models define the different levels of cloud services provided to users. Depending on the level of control and responsibility required, cloud services are categorized into three main models:

- Infrastructure as a Service (IaaS)
- Platform as a Service (PaaS)
- Software as a Service (SaaS)

---

## 1. Infrastructure as a Service (IaaS)

**Definition**

Infrastructure as a Service (IaaS) provides virtualized computing resources such as virtual machines, storage, networking, and servers over the internet. Instead of purchasing and maintaining physical hardware, users rent these resources on demand.

### Key Features

- Virtual machines and servers
- Scalable storage
- Networking services
- Pay-as-you-go pricing
- High flexibility and control

### Advantages

- Eliminates the cost of purchasing hardware.
- Resources can be scaled up or down easily.
- Suitable for testing, development, and hosting applications.
- Provides complete control over the operating system and software.

### Limitations

- Users are responsible for managing operating systems and applications.
- Requires technical knowledge to configure and maintain resources.

### Examples

- Amazon EC2
- Microsoft Azure Virtual Machines
- Google Compute Engine

---

## 2. Platform as a Service (PaaS)

**Definition**

Platform as a Service (PaaS) provides a complete development environment where developers can build, test, deploy, and manage applications without worrying about the underlying infrastructure.

### Key Features

- Ready-to-use development environment
- Built-in development tools
- Automatic software updates
- Database and application hosting
- Supports team collaboration

### Advantages

- Faster application development.
- No need to manage servers or operating systems.
- Reduces development time and maintenance effort.
- Developers can focus on writing code.

### Limitations

- Less control over the underlying infrastructure.
- Platform-specific limitations may exist.

### Examples

- Microsoft Azure App Service
- Google App Engine
- Heroku

---

## 3. Software as a Service (SaaS)

**Definition**

Software as a Service (SaaS) delivers fully functional software applications over the internet. Users simply access the software through a web browser without installing or maintaining it.

### Key Features

- Accessible through a web browser
- No installation required
- Automatic updates
- Subscription or pay-per-use model
- Easy accessibility from any device

### Advantages

- Easy to use and deploy.
- No maintenance or software installation required.
- Accessible from anywhere with an internet connection.
- Automatic updates and backups.

### Limitations

- Limited customization.
- Requires an internet connection.
- Users have less control over the software.

### Examples

- Gmail
- Microsoft 365
- Google Docs
- Zoom
- Dropbox

---

## Key Takeaways

- IaaS provides virtual infrastructure and offers the highest level of control.
- PaaS provides a platform for developers to build and deploy applications.
- SaaS delivers ready-to-use software that users can access directly through the internet.
- Choosing the right service model depends on the level of control, flexibility, and management required.

«Simple way to remember:

- IaaS → Rent Infrastructure
- PaaS → Rent a Platform
- SaaS → Use Software»
