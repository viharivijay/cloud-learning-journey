# Cloud Security Fundamentals

**Learning Objectives**

- Understand the importance of cloud security.
- Learn the core principles of information security.
- Explore common security threats and protection mechanisms in cloud computing.

## What is Cloud Security?
Cloud security is a collection of technologies, policies, controls, and best practices used to protect cloud infrastructure, applications, and data from cyber threats and unauthorized access.

## Core Security Principles (CIA Triad)

### 1. Confidentiality

Ensures that only authorized users can access sensitive information.

**Examples:**
- Encryption
- Multi-Factor Authentication (MFA)
- Access Control
- 
### 2. Integrity
Ensures that data remains accurate, complete, and unaltered.

**Examples:**
- Hashing
- Digital Signatures
- Data Validation

### 3. Availability
Ensures that systems and data are accessible whenever needed.

**Examples:**
- Backup and Recovery
- Load Balancing
- Disaster Recovery
  
## Common Cloud Security Threats

- Data breaches
- Malware and ransomware
- Phishing attacks
- Weak passwords
- Insider threats
- DDoS (Distributed Denial of Service) attacks

## Cloud Security Best Practices

- Use Multi-Factor Authentication (MFA).
- Encrypt sensitive data.
- Apply the Principle of Least Privilege (PoLP).
- Regularly update and patch systems.
- Monitor logs and security events.
- Perform regular backups.

## Shared Responsibility Model

In cloud computing, security responsibilities are shared:

- **Cloud Provider:** Secures the cloud infrastructure (hardware, networking, physical data centers).
- **Customer:** Secures data, user accounts, applications, and access permissions.

## Key Takeaways
- Cloud security protects cloud resources from cyber threats.
- The **CIA Triad** (Confidentiality, Integrity, Availability) is the foundation of information security.
- Both the cloud provider and the customer have security responsibilities.
- Following best practices significantly reduces security risks.
