# Day 12 – Cloud Monitoring, Logging & Cost Optimization

## Overview
Cloud monitoring continuously tracks the health, availability, and performance of cloud resources. Logging records system events and activities, helping organizations troubleshoot issues, improve security, and maintain compliance.

## Key Concepts

### Cloud Monitoring
- Tracks CPU, memory, storage, and network performance.
- Detects failures and performance bottlenecks.
- Improves system reliability.

### Cloud Logging
- Stores records of application and system events.
- Helps in debugging, auditing, and security analysis.

### Alerts
- Trigger notifications when predefined thresholds are exceeded.
- Enable quick response to system failures.

### Cost Optimization
- Choose the right resource size.
- Use Auto Scaling.
- Delete idle resources.
- Select appropriate pricing models.
- Monitor cloud spending regularly.

## Popular Monitoring Services
- AWS CloudWatch
- Azure Monitor
- Google Cloud Monitoring

## Key Takeaways
- Monitoring ensures system reliability.
- Logging helps identify and resolve issues.
- Alerts enable proactive maintenance.
- Cost optimization reduces unnecessary cloud expenses while maintaining performance.
