# Day 13 – Cloud Automation & Infrastructure as Code (IaC)

## Overview

Today's learning focused on Infrastructure as Code (IaC), a modern approach to managing cloud resources through code instead of manual configuration. IaC enables faster deployments, consistent environments, and improved scalability.

## Topics Covered

- What is Infrastructure as Code (IaC)?
- Benefits of IaC
- Traditional Deployment vs IaC
- Popular IaC Tools
  - Terraform
  - AWS CloudFormation
  - Azure ARM Templates
  - Azure Bicep
  - Google Cloud Deployment Manager
- Declarative vs Imperative Approaches
- Immutable Infrastructure
- Infrastructure Lifecycle
- IaC Best Practices
- Real-world Use Cases

## Key Learnings

- Infrastructure can be provisioned automatically using code.
- IaC improves consistency, automation, and scalability.
- Terraform is a leading multi-cloud IaC tool.
- Version control with GitHub makes infrastructure changes traceable and collaborative.
- Adopting IaC reduces manual effort and minimizes configuration errors.

## Outcome

Completed an introduction to Infrastructure as Code and learned how automation improves cloud infrastructure management and prepares the foundation for DevOps and cloud engineering practices.
